# Step 3: Git & GitHub Workflow

## Overview

Learn version control with Git and collaborative development with GitHub. This step is practiced through working on all other projects using proper branching, commits, and pull requests.

## Learning Goals

- Understand Git fundamentals (add, commit, push, pull)
- Work with branches effectively
- Create and manage Pull Requests
- Write meaningful commit messages
- Handle merge conflicts

## Requirements

### Git Basics
- [ ] Initialize and clone repositories
- [ ] Stage and commit changes with clear messages
- [ ] Push to and pull from remote

### Branching
- [ ] Create feature branches from main
- [ ] Switch between branches
- [ ] Merge branches locally

### Pull Requests
- [ ] Open draft PRs early for visibility
- [ ] Request reviews and address feedback
- [ ] Merge PRs after approval

### Commit Messages
Follow conventional commit format:
```
feat: add new feature
fix: resolve bug
refactor: improve code structure
docs: update documentation
style: formatting changes
```
