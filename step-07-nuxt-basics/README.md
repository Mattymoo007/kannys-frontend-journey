# Step 7: Nuxt 3 Basics

## Overview

Convert the To-Do app to a Nuxt 3 application. Learn server-side rendering, file-based routing, and the Nuxt ecosystem.

## Prerequisites

- Completed Step 6 (Pinia State Management)

## Learning Goals

- Understand SSR vs SPA vs SSG
- Set up a Nuxt 3 project
- Use file-based routing
- Configure Nuxt modules

## Requirements

### Project Setup
- [ ] Create new Nuxt 3 project
- [ ] Migrate components from Vue app
- [ ] Configure Nuxt with Pinia module

### File-Based Routing
- [ ] Create pages: index, tasks, about
- [ ] Implement dynamic routes for task details
- [ ] Add navigation between pages

### Nuxt Features
- [ ] Use `useFetch` or `useAsyncData` for data fetching
- [ ] Configure `nuxt.config.ts` properly
- [ ] Use auto-imports for composables and components

### Layouts
- [ ] Create default layout with navigation
- [ ] Add error page

## Success Criteria

- [ ] App works as Nuxt application
- [ ] Multiple pages with proper routing
- [ ] Navigation works correctly
- [ ] State persists across page navigation
